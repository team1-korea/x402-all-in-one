# Snowman Sabotage — 랜딩 화면 개선 Design Spec

**Date:** 2026-05-16
**Scope:** `SnowmanSabotageQuest.tsx` > `LandingScreen` 컴포넌트만 변경

---

## 1. 목적

블록체인 비전문가가 게임을 시작하기 전에 "무엇을, 왜, 어떻게" 하는지 직관적으로 이해할 수 있도록 랜딩 화면을 개선한다. 기존 랜딩은 Slush 파라미터(K=5, α=3)를 바로 노출해 진입 장벽이 높다.

---

## 2. 최종 확정 디자인

### 2-1. 레이아웃 구조 (Option C)

```
[Quest 배지]
[제목: 아발란체 합의를 방해하라!]

[설명 블록 — 내러티브 한 문단]
  블록체인 개념 → 아발란체 합의 방식 → 유저 역할 순서로 흐르는 텍스트

[데모 영역]
  ▸ 이렇게 클릭하면 됩니다
  [2×2 눈사람 그리드] + [JS 커서 자동 애니메이션]

[CTA 버튼: 합의 방해하러 가기 →]
```

### 2-2. 설명 블록 텍스트

```
블록체인이 작동하려면 수백 개의 컴퓨터(노드)가 "이 거래가 맞다"고 동의해야 합니다.
아발란체는 각 노드가 무작위로 이웃 5개에게 물어보고,
다수가 동의하면 자신도 따르는 방식으로 빠르게 합의합니다.

당신의 역할: 파란 눈사람(정직한 노드)을 클릭해 빨간 눈사람(악의적 노드)으로 바꾸세요.
합의가 완료되기 전에 다수를 장악하면 됩니다. 총 3번의 블록 생성을 방해해보세요.

확률적으로 거의 불가능합니다. 건투를 빕니다.
```

### 2-3. 데모 영역 — 눈사람 배치

눈사람 4개를 **2행 2열 그리드**로 배치. 노드3만 파란색 유지, 나머지 3개가 순서대로 클릭됨.

```
[노드1] [노드2]
[노드3] [노드4]
```

커서 클릭 순서: 노드1 → 노드4 → 노드2 → 전체 리셋 → 반복.
노드3은 클릭되지 않아 "파란 노드가 버텨있다"는 시각적 대비 효과.

## 3. 커서 애니메이션 구현

### 3-1. 방식

CSS keyframes 하드코딩 방식 **사용 안 함**.
JS로 각 눈사람 SVG의 실제 DOM 위치를 `getBoundingClientRect()`로 측정해 커서를 정확히 이동시킨다.
창 크기가 달라져도 항상 정확히 눈사람 머리 위에 위치.

### 3-2. 커서 이동

```
CSS transition: left/top 0.55s cubic-bezier(0.25, 0.46, 0.45, 0.94)
타겟 좌표: SVG 내 머리 중심 (cy=17, r=10) → 실제 렌더 좌표로 변환
```

### 3-3. 클릭 인터랙션 (두 가지 동시 발동)

**커서 스쿼시:**
```css
@keyframes cursorClick {
  0%   { transform: scale(1)    rotate(0deg); }
  30%  { transform: scale(0.72) rotate(-8deg); }   /* 눌리는 느낌 */
  70%  { transform: scale(1.18) rotate(4deg); }    /* 튕겨나오는 느낌 */
  100% { transform: scale(1)    rotate(0deg); }
}
duration: 0.22s ease-out
```

**오렌지 리플:**
```css
@keyframes ripplePop {
  0%   { opacity: 0.7; transform: scale(0.3); }
  60%  { opacity: 0.3; transform: scale(1.2); }
  100% { opacity: 0;   transform: scale(1.6); }
}
duration: 0.45s ease-out
크기: 32×32px, border-radius: 50%, color: rgba(251,146,60,0.5)
```

### 3-4. 루프 타이밍

```
이동 transition: 550ms
클릭 + 리플:    450ms
클릭 후 대기:   650ms
루프 간 대기:   1000ms
```

### 3-5. 눈사람 색 전환

클릭 시 해당 `.snowman`에 `.red` 클래스 추가:
```css
.snow-body { fill: #4ECDC4; transition: fill 0.35s ease; }
.snowman.red .snow-body { fill: #FF6B6B; }
```

루프 리셋 시 `.red` 클래스 제거.

## 5. 범위 외

- 게임 화면 내 튜토리얼 오버레이
- 랜딩 화면에서 직접 눈사람을 클릭 가능하게 하는 인터랙티브 프리뷰
- Slush 파라미터(K, α) 설명 노출 (현재 랜딩에서 제거, 게임 화면에는 유지 가능)
