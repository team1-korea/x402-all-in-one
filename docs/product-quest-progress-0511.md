## 📌 1. 현재까지 진행된 내용 (수정 사항)

### 💡 기획 및 규칙 확정
- 구조: 상품별로 독립된 10단계 퀘스트 경로 (무료 1개 + 유료 9개)
- 규칙:
  - 1인 1경로: 동시 참여 불가.
  - 1인 1상품 제한: 한 사람이 여러 상품을 독식하는 것 방지.
  - 선점 경쟁: 누군가 먼저 상품을 획득하면 해당 경로는 종료되며, 다른 사용자는 재시작.
- 결제: 유료 퀘스트(#2~#10) 진행 시 x402 (EIP-3009 기반 TONE 토큰) 결제 사용. (가스비는 APIX 사용)

### 🛠️ 스킬(클라이언트) 싱크 완료
- 작업 위치: `x402-all-in-one` 레포지토리의 `feature/journey0511` 브랜치
- quest/SKILL.md 수정:
  - 엔드포인트를 상품별 독립 경로인 `/v1/quest/{productId}/{step}` 형태로 동적화.
  - 10단계 퀴즈 유형(OX, 객관식, 비밀코드) 및 가격/보상 밸런싱 초안 작성.
  - 10번 퀘스트에 도달 시 클로드가 사용자에게 웹 접속을 유도하는 가이드라인 추가.
- discover/SKILL.md 수정:
  - 화폐 단위를 APIX에서 결제 토큰인 TONE으로 수정하여 혼선 방지.

### 🖥️ 서버 사이드 (`x402-server`)
1. 상품별 라우팅 및 데이터 구조 구현:
   - `/v1/quest/{productId}/{step}` 요청을 처리할 수 있도록 서버 라우터 수정.
2. 비밀코드(Secret Code) 검증 로직:
   - 10번 퀘스트의 정답(특정 문자열)을 검증하는 로직 추가.
3. 상품별 퀘스트 데이터 구성:
   - `quests.ts`에 `product-a`를 위한 10단계 문제 세트 구성 완료.
4. 지갑 주소 기반 상태 관리 구현:
   - 사용자의 번거로움을 줄이기 위해 지갑 주소를 식별자로 사용.
   - `db.ts`에 사용자의 현재 진행 단계(`currentStep`)와 선택한 상품(`currentProductId`)을 기록하는 필드 추가.
   - 라우터에서 이를 검증하여 순차 진행 및 1인 1경로 규칙을 강제함.
