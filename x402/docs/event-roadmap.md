# 🏆 완벽한 이벤트를 위한 최종 로드맵 (Event Roadmap)

본 문서는 x402-escape 프레임워크를 활용하여 **성공적인 참가자 경험**과 **매끄러운 운영**을 달성하기 위한 단계별 실행 리스트(TODO)입니다.

## ✅ Phase 0. 인프라 및 코어 기강 확립 (완료)
본 프레임워크의 근간이 되는 핵심 엔진과 보안 체계가 구축되었습니다.

- [x] **계층형 아키텍처 정립**: Core, App API, CLI, Runtime 데이터 레이어 분리.
- [x] **X402 결제 게이트 하드닝**: 온체인 TxHash 검증 및 리플레이 공격 방어 로직 구현.
- [x] **원자적 상태 관리 엔진**: `withFileLock` 기반 동시성 제어 및 데이터 무결성 보장.
- [x] **운영 자동화 도구**: CLI(`init`, `seed`)를 통한 이벤트 리셋 및 배포 자동화.
- [x] **Docker 운영 환경**: 4000번 포트 및 독립 볼륨 기반의 Production-Ready 환경 구축.
- [x] **Fail-Fast UX 세이프가드**: 플레이 중 실시간 재고 체크 기반 조기 이탈 방지 정책 적용.

## 🏁 Phase 1. 프리미엄 참가자 온보딩 (Web UI)
가장 먼저 참가자가 자신의 정보를 입력하고 게임에 진입할 '대문'이 필요합니다.

- [ ] **참가자 등록 웹페이지 구현**:
  - [ ] 닉네임, 이메일, 관심사 등 네트워킹용 메타데이터 수집 폼 제작.
  - [ ] 입력된 정보를 백엔드 `/api/products` (POST)와 연동하여 자동으로 세션 생성.

## 🤝 Phase 2. 네트워킹 이벤트 설계 (Dynamic Quiz)
참가자들이 서로 소통하게 만드는 '장치'를 퀴즈에 녹여냅니다.

- [ ] **실시간 참가자 정보 기반 퀴즈 엔진**:
  - [ ] 수집된 유저 DB를 기반으로 "지금 제일 먼저 입장한 사람의 닉네임은?" 등의 가변형 퀴즈 자동 생성.
  - [ ] '서로 지갑 주소 교환하기' 미션을 통한 네트워킹 유도.
- [ ] **퀴즈 난이도 밸런싱**:
  - [ ] 1~3단계: 입문용 (Team1 퀴즈).
  - [ ] 4~7단계: 네트워킹용 (동료 찾기 퀴즈).
  - [ ] 8~10단계: 기술/심화용 (APIX L1 지식 퀴즈).
