# 📋 운영 매뉴얼: x402-Escape 이벤트 가이드

x402-escape 이벤트를 성공적으로 개최하고 운영하기 위한 실무 지침서입니다.

---

## 1. 환경 설정 (.env)
프로덕션 환경을 위해 루트 디렉토리에 필수적으로 정의되어야 하는 변수입니다.
- `RPC_URL`: APIX L1 네트워크 노드 엔드포인트.
- `GATEWAY_ADDRESS`: 참가자들의 x402 결제를 수신할 마스터 운영 지갑.
- `ADMIN_PRIVATE_KEY`: 참가자들에게 퀴즈 보상을 자동 전송할 관리자 개인키 (APIX 잔액 필수).
- `API_URL`: 참가자 CLI가 접속할 백엔드 서버의 퍼블릭 주소.

## 2. CLI 주요 명령어 (Operations)
- `npx x402-escape init`: 필수 디렉토리 구조 및 초기 JSON 파일 생성.
- `npx x402-escape seed`: **팩토리 리셋**. 모든 런타임 데이터를 초기 템플릿 상태로 복구.
- `npx x402-escape validate`: 시스템 환경 변수 결합도 및 RPC 연결 상태 실시간 진단.
- `npx x402-escape status`: 실시간 상품 재고 및 당첨자 현황 대시보드 출력.

## 3. 참가자 온보딩 가이드
- `npx x402-escape skill`: 참가자들을 위한 Claude Code 스킬 설정 가이드를 출력합니다.
- `npx x402-escape install-skill`: 참가자의 로컬 환경에 필요한 스킬 파일을 자동으로 설치합니다.
- **Seed Faucet**: 참가자가 처음 시스템에 진입할 때 자동으로 **1 APIX**를 지급하여 첫 트랜잭션 가스비와 결제 비용을 보조합니다.

## 4. 운영 및 리셋 정책
- **경로 리셋 (Reset Policy)**: 특정 상품의 재고가 0이 되는 즉시, 해당 상품을 목표로 하던 모든 참가자(미완료자)는 자동으로 '상품 선택' 단계로 리셋됩니다.
- **보안 기강**: 관리자 개인키는 절대 로깅되지 않으며, 배포 전 `runtime/` 내의 테스트 데이터와 터미널 히스토리를 반드시 정화해야 합니다.

## 5. 비상 조치 가이드
- **데이터 오염 시**: `seed` 명령어를 실행하여 데이터를 초기화합니다. (주의: 참가자들의 기존 진행 상태가 사라집니다.)
- **RPC 연결 장애 시**: `.env`의 `RPC_URL`을 체크하고 백엔드 서비스를 재시작합니다.
