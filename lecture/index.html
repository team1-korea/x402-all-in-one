05 · Theory — 아이폰 비유
아이폰 사러 애플스토어 가기

아 아이폰 사고 싶다 → 필요한 게 생겼다
2

3

4
"결제해주세요" → 얼마를, 어디로, 언제까지
5
카드 꽂기 → 서명 생성 + 다시 요청
6
카드사 승인 → 검증 + 블록체인 정산
7
아이폰 수령 → 서비스 응답

#구멍힌트
①몇 번 응답?이 프로토콜 이름이기도 한 HTTP 상태 코드
②응답에 뭐가 있나?체인, 금액, 수신 지갑 — "가격표 + 계좌번호"
③to 필드?accepts[0]의 수신자 주소 필드명
④value 필드?accepts[0]의 금액 필드명
⑤validBefore?지금 + maxTimeoutSeconds초
⑥인코딩 방식?JSON → 바이너리 → 텍스트
⑦누가 검증?네트워크 정보 테이블에 있음
정확하지 않아도 됩니다 — 비슷한 의미면 Claude가 알아서 해석합니다
